# 프로젝트 청사진: 전국 도시/군 계획 조례 검색

## 1. 개요

이 프로젝트의 목표는 사용자가 대한민국 전체의 특정 지역(도, 시, 군, 구)을 선택하고, **법령정보센터 오픈 API 키**와 검색어를 입력하여 해당 지역의 '도시계획 조례' 또는 '군계획 조례'를 **직접 조회하고 결과를 앱 내에서 확인**할 수 있는 웹 애플리케이션을 구축하는 것입니다.

## 2. 기능 및 디자인 명세

### 가. 핵심 기능

1.  **지역 선택 UI**:
    *   2단계 드롭다운 메뉴를 통해 광역자치단체와 기초자치단체를 순서대로 선택합니다.

2.  **검색어 입력**:
    *   사용자가 '발전시설', '용적률' 등 조례에서 찾고 싶은 특정 키워드를 입력할 수 있습니다.

3.  **API 키 입력**:
    *   사용자가 발급받은 **법령정보센터 오픈 API 인증키**를 입력할 수 있는 비밀번호 필드를 제공하여 안전하게 키를 관리합니다.

4.  **API 검색 및 결과 표시**:
    *   '검색' 버튼을 클릭하면, 선택된 지역, 키워드, API 키를 조합하여 법령정보센터 API에 직접 요청을 보냅니다.
    *   API로부터 받은 검색 결과를 **페이지 내에 동적으로 생성**하여 목록 형태로 보여줍니다. 각 항목은 조례명, 공포/시행일자, 소관부처 정보를 포함하며, 원문 링크를 제공합니다.
    *   검색 결과가 없을 경우, 적절한 메시지를 표시합니다.

5.  **페이지네이션**:
    *   검색 결과가 많을 경우를 대비하여 '이전'과 '다음' 버튼을 포함한 간단한 페이지네이션 기능을 제공하여 사용자 경험을 개선합니다.

### 나. 디자인 및 스타일

*   **레이아웃**: 사용자가 기능에 집중할 수 있도록 중앙에 검색 폼을 배치하고, 그 아래에 검색 결과 영역을 동적으로 표시합니다.
*   **색상**: 신뢰감을 주는 파란색 계열을 주 색상으로 사용합니다.
*   **폰트**: 가독성이 높은 'Noto Sans KR' 폰트를 사용합니다.
*   **결과 표시**: 검색 결과는 깔끔한 카드 형태로 디자인하여 가독성을 높입니다. 각 카드에는 조례명, 상세 정보, 그리고 원문으로 바로 갈 수 있는 링크가 포함됩니다.
*   **반응형 디자인**: 데스크톱과 모바일 기기 모두에서 최적화된 화면을 볼 수 있도록 반응형으로 디자인합니다.

## 3. 기술 스택

*   **HTML**: 웹페이지의 구조를 정의합니다. (API 키 입력 필드, 결과 컨테이너 추가)
*   **CSS**: 시각적 스타일과 레이아웃을 담당합니다. (결과 목록, 페이지네이션 스타일 추가)
*   **JavaScript (ES Modules)**:
    *   행정구역 데이터 관리 및 드롭다운 메뉴 동적 업데이트.
    *   **`fetch` API**를 사용한 비동기 API 호출.
    *   JSON 데이터 파싱 및 동적 HTML 생성을 통한 결과 표시.
    *   페이지네이션 로직 구현.

## 4. 구현 현황

1.  **`index.html`**:
    *   기본적인 HTML 구조 위에 API 키 입력을 위한 `<input type="password">` 필드를 추가했습니다.
    *   검색 결과를 표시할 `<div>` 컨테이너(`results-container`)와 목록(`results-list`), 그리고 페이지네이션을 위한 영역(`pagination`)을 추가했습니다.

2.  **`style.css`**:
    *   새로 추가된 API 키 필드, 결과 컨테이너, 결과 목록, 페이지네이션 버튼 등에 대한 스타일을 추가하여 전체적인 디자인 통일성을 유지했습니다.

3.  **`main.js`**:
    *   기존의 외부 사이트 리디렉션 로직을 **완전히 제거**했습니다.
    *   '검색' 버튼 클릭 시 `fetch`를 사용하여 법령정보센터 API(`www.law.go.kr/DRF/lawSearch.do`)를 호출하는 `searchOrdinances` 함수를 구현했습니다.
    *   API 응답을 받아 `displayResults` 함수를 통해 검색 결과를 동적으로 HTML 리스트 아이템으로 생성하고, `displayPagination` 함수를 통해 페이지네이션을 구현했습니다.
    *   API 키 누락, 검색 결과 없음, 네트워크 오류 등 다양한 시나리오에 대한 에러 처리 로직을 추가했습니다.
